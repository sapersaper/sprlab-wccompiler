## 🎯 What Was the Problem?

The WCC compiler was generating code that called wrapper methods (`_username()`, `_age()`, etc.) but **never created them**, causing runtime errors:

```
ReferenceError: this._username is not a function
```

This made **two-way data binding completely unusable** with `defineModel`.

### Example of Broken Code (Before Fix):

```javascript
// Generated by compiler (BROKEN):
this.__model_username_0.value = this._username() ?? '';  // ❌ Method doesn't exist!

this.__model_username_0.addEventListener('input', (e) => { 
  this._username(e.target.value);  // ❌ Method doesn't exist!
});
```

---

## ✅ What Was Fixed?

Added **wrapper method generation** in the compiler for every `defineModel` declaration.

### Generated Code (After Fix):

```javascript
// For: const username = defineModel({ name: 'username', default: '' })

// Internal signal (already existed)
this._m_username = __signal('');

// NEW: Wrapper method (dual getter/setter)
_username(val) {
  if (arguments.length === 0) {
    return this._m_username();  // Getter mode
  } else {
    this._modelSet_username(val);  // Setter mode (dispatches events)
  }
}

// Now template code works correctly:
this.__model_username_0.value = this._username() ?? '';  // ✅ Works!

this.__model_username_0.addEventListener('input', (e) => { 
  this._username(e.target.value);  // ✅ Works!
});
```

### 1. Core Implementation
**File**: `lib/codegen.js`  
**Lines**: 1958-1974  
**Changes**: Added wrapper method generation logic (+18 lines)

```javascript
// Wrapper methods for defineModel signals (dual getter/setter)
if (modelDefs.length > 0) {
  lines.push('  // --- Model wrapper methods ---');
  for (const md of modelDefs) {
    lines.push(`  _${md.name}(val) {`);
    lines.push(`    if (arguments.length === 0) {`);
    lines.push(`      return this._m_${md.name}();`);
    lines.push(`    } else {`);
    lines.push(`      this._modelSet_${md.name}(val);`);
    lines.push(`    }`);
    lines.push(`  }`);
    lines.push('');
  }
}
```
