## Description
Two property-based tests in the VSCode language server are failing consistently, blocking the release pipeline. These tests use `fast-check` for property-based testing and are failing with specific counterexamples.

## Failing Tests

### Test 1: "mapping covers the entire block content range"
**Location**: `vscode-wcc/packages/server/src/languagePlugin.property.test.ts` line ~214

**Error**:
```
Property failed after 51 tests
Counterexample: [{"scriptLang":"","scriptContent":" ","templateContent":"<A","styleContent":" "}]
AssertionError: expected 1 to be 5 // Object.is equality
```

**Expected**: Mapping should cover entire block content range (length 5)
**Actual**: Only covers length 1

### Test 2: "after update(), embeddedCodes reflect the modified content, not the initial content"
**Location**: `vscode-wcc/packages/server/src/languagePlugin.property.test.ts` line ~277

**Error**:
```
Property failed after 93 tests
Counterexample: [{"scriptLang":"","initialScript":"","initialTemplate":"","initialStyle":"","modifiedScript":"","modifiedTemplate":"<A","modifiedStyle":"","includeScript":true,"includeTemplate":true,"includeStyle":false}]
AssertionError: expected '\nA;\n' to be '' // Object.is equality
```

**Expected**: Empty string after update
**Actual**: Contains `"\nA;\n"`

## Root Cause Analysis Needed

The tests are property-based using `fast-check`, which means they:
1. Generate random inputs automatically
2. Find minimal counterexamples that break the property
3. Are deterministic once a counterexample is found

The counterexamples suggest issues with:
1. **Test 1**: Position mapping calculation when template content is minimal (`"<A"`)
2. **Test 2**: Content update logic leaving residual content (`"\nA;\n"`)

## Resolution

### Root Cause
The script block's embedded code was including a `usageSuffix` (generated content to suppress "declared but never read" warnings) in the snapshot, but the source mapping only covered the original block content. This caused a mismatch where:
- Snapshot length = block.content + usageSuffix
- Mapping length = block.content only

### Fix Applied
Removed the `usageSuffix` generation logic from `languagePlugin.ts` because it was redundant. The template variables are already available in the `template_expressions_0` virtual code through the script prefix, making the suffix unnecessary.

**File modified**: `vscode-wcc/packages/server/src/languagePlugin.ts`
- Removed lines 268-295 (usageSuffix generation)
- Updated line 290 to use only `exposeSuffix`

### Test Results
- ✅ All 12 tests in `languagePlugin.property.test.ts` now pass
- ✅ Full test suite: 981/982 tests passing
- ❌ Only 1 pre-existing failure (e2e/playwright missing dependency)

### Verification
