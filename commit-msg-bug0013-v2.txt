fix(template-normalizer): pre-process Mustache key bindings before HTML parsing (BUG-0013 v2)

ROOT CAUSE:
- QA used key={{ item.id }} WITHOUT quotes
- Previous fix only handled key="{{ expr }}" WITH quotes
- HTML parser broke unquoted Mustache into malformed attributes
- Components failed to render (innerHTML = 0 bytes)
- Console showed syntax errors and bind() failures

SOLUTION:
- Added pre-processing in template-normalizer.js BEFORE HTML parser sees template
- Two regex patterns convert both quoted and unquoted Mustache to :key syntax
- Prevents HTML parser from breaking Mustache expressions
- Maintains keyed reconciliation functionality

CODE CHANGES:
- lib/template-normalizer.js: Added Mustache-to-binding conversion
  - Pattern 1: with quotes
  - Pattern 2: without quotes
  - Runs before PascalCase conversion and self-closing tag expansion

TESTING:
- All key-binding tests passing
- 1046/1046 total tests passing (100% pass rate)
- No regressions detected
- Generated code shows clean templates instead of malformed attributes
- Keyed reconciliation working correctly
